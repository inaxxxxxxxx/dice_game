# CLAUDE.md

このファイルは Claude Code がこのリポジトリで作業する際の前提情報です。

## プロジェクト概要

チンチロ（3つの骰子を使った賭けゲーム）をブラウザで遊べる Web アプリ。
ビルドツールなし、フレームワークなしの素の HTML/CSS/JS。骰子は Three.js (r128, CDN読み込み) で3D表示。

## ファイルの役割

- `index.html` — 全画面分の DOM をあらかじめ用意し、`hidden` 属性で表示切り替えする SPA 構成
- `style.css` — モダン・ミニマル（白基調、明朝体の見出し＋サンセリフの本文）。CSS変数で配色管理
- `game.js` — ゲーム状態管理（`state` オブジェクト）、役判定（`judgeHand`）、勝敗判定（`resolveOutcome`）、画面遷移
- `dice3d.js` — Three.js シーン構築、疑似物理（重力・反発・摩擦）、出目読み取り（`readTopFace`）

## 重要な設計判断（変更時に注意）

### 役の強さの序列
`game.js` の `HAND_RANK` で定義。伝統ルールに基づき以下の順（強い→弱い）：
ピンゾロ（1-1-1、最強）＞ アラシ（ゾロ目）＞ シゴロ（4-5-6）＞ 通常の目（2つ同じ＋1つ、数字の大小で比較）＞ 目なし ＞ ヒフミ（1-2-3、反則目で最弱・即負け）

`compareHands()` で比較する。アラシと通常の目は `subValue`（数字の大小）でサブ比較が必要、ピンゾロ・シゴロ・ヒフミ・目なしは同ランク内で常に同等。

### 勝敗確定とサドンデスのロジック（`resolveOutcome`）
- 出目を強い順にグループ化（同ランクは同着グループ）
- 弱い方から負け確定人数の枠を埋めていく
- グループが枠に対して大きすぎる（同着で枠を奪い合う）場合、そのグループが「サドンデス対象」になり、**それより強い全グループは即座に勝ち抜け確定**にする（ここを間違えると、サドンデス対象より強いはずのプレイヤーが誤って負け扱いになるバグが起きるので注意）
- サドンデス対象者だけで `state.activeIndices` を絞り、`beginRound()` を再帰的に呼ぶ
- サドンデスが何重に発生しても対応できる設計（同着が続く限り再戦を繰り返す）

### `state.finalRecord` の役割
`state.results` はラウンドごとに `beginRound()` でリセットされるため、サドンデス前に確定したプレイヤーの出目・役を最終結果画面で表示するには別途スナップショットが必要。`decidedWinners` / `decidedLosers` に追加するタイミングで必ず `state.finalRecord[idx] = state.results[idx]` も書き込むこと。これを忘れると最終結果画面で `undefined` 参照エラーになる。

### 3D骰子の面とテクスチャの対応
`dice3d.js` の `FACE_VALUES = [1, 6, 2, 5, 3, 4]` は Three.js の `BoxGeometry` のマテリアル順（+x, -x, +y, -y, +z, -z）に対応。対面の合計が7になる標準サイコロの規則を満たす（1-6, 2-5, 3-4）。`readTopFace()` も同じ順序の法線配列を使っており、これがズレると出目とビジュアルが一致しなくなる。

## 動作確認の方法

ブラウザ実機でのスクリーンショット確認ができない環境では、jsdom + モック化した `Dice3D.onSettled` でゲームフローと判定ロジックをシミュレーションするのが有効（このリポジトリにテストファイルは残していないが、必要なら `npm install jsdom` して `dom.window.eval(gameJsSource)` でロードし、ボタンの `.click()` を叩いて検証できる）。
